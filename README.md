# TOOCK Backend

## 1. 개요
- **프로젝트 배경:** 취업 준비생들이 면접 기출을 암기해도 실제 말하기 연습이 부족하다는 문제를 해결하고자 함.
- **문제 정의:** 접근성 높은 면접 연습 플랫폼의 부재.
- **프로젝트 목표:** IT 직군 일부를 대상으로 면접 질문 생성, 음성 기반 응답 수집, AI 피드백을 제공하는 면접 대비 서비스를 개발.
- **기대 효과:** 면접 외적 요소 없이 연습 환경 제공, AI 피드백을 통한 보완점 확인, 기업 데이터 기반 맞춤형 면접 대비 가능.

## 2. 기술 스택 및 개발 환경
### Backend
- **언어/프레임워크:** Java 17, Spring Boot
- **인증:** OAuth 2.0 (Google, Kakao)
- **데이터베이스:** MySQL (AWS RDS)
- **ORM:** Spring Data JPA

### AI/ML 기능
- 면접 질문/답변 생성: Google Gemini API
- 음성 → 텍스트 변환: OpenAI Whisper API
- 외부 데이터 수집: Python 크롤링 모듈

### Infra / DevOps
- **서버 인프라:** AWS EC2, Nginx 리버스 프록시
- **DB 인프라:** AWS RDS (MySQL)
- **스토리지:** AWS S3
- **배포 자동화:** GitHub Actions → EC2 배포

### Frontend
- **프레임워크:** Next.js
- **상태 관리:** TanStack Query, Zustand
- **HTTP Client:** ky
- **UI/Animation:** clsx, framer-motion
- **IDE/배포:** VS Code, Vercel

> 현재 리포지토리는 백엔드 코드 전용이며, 프론트엔드 리포지토리는 별도 운영됩니다. 두 코드베이스를 함께 보려면 각 리포지토리를 개별적으로 확인해야 합니다.

## 3. 시스템 설계
- Spring Boot 기반 API 서버가 인증, 면접 세션 관리, AI 연동을 담당합니다.
- AI 서비스(Gemini, Whisper)와의 연동으로 질문 생성, 음성 인식, 피드백 생성 흐름을 구성합니다.
- AWS RDS(MySQL)로 면접 이력, 사용자 정보 등을 저장하며, S3에 음성/미디어 자산을 저장합니다.

## 4. 구현 내용
- 음성 기반 면접 진행: 사용자 음성 녹음을 수신하고 Whisper로 STT 처리.
- 질문/답변 생성 및 피드백: 기업/직무별 설정에 따라 Gemini로 질문을 생성하고 답변 평가를 진행.
- OAuth 2.0 인증: Google/Kakao 로그인 지원.

## 5. 결과 및 향후 개선 방향
- **성과:** 실제 면접 환경과 유사한 음성 인터랙션 및 AI 피드백 제공으로 목표 달성.
- **한계:** AI 피드백 정밀도와 점수의 객관성 개선 필요, 실무형 질문 강화를 위한 프롬프팅 보완 필요.
- **다음 단계:** 평가 기준 고도화, 프론트엔드/백엔드 연동 안정화, 추가 직군 커버리지 확장.
